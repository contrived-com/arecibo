# Claudia Progress Log

This file tracks completed work by the Claudia autonomous agent.
Each entry documents what was implemented and lessons learned.

## Codebase Patterns

- Rust toolchain: use `RUSTUP_HOME=$HOME/.rustup CARGO_HOME=$HOME/.cargo` in CI containers where `/usr/local/cargo` is read-only
- Transponder binary name: `transponder` (declared in Cargo.toml [[bin]])
- HTTP client: uses `ureq` 2.x (blocking, lightweight) instead of `reqwest` to avoid async/tokio overhead
- Thread-safe counters: `Arc<Mutex<TransponderCounters>>` shared between runtime and ingest threads; locks are never held during HTTP calls
- Ingest socket: background thread with 500ms read timeout for clean shutdown (matches Python socket.close() pattern)
- Config env parsing: `env_bool` for truthy values, `env_int` with minimum enforcement, float timeout parsing with defaults
- Vault fallback: synchronous ureq calls with 2s timeout, only runs once at startup
- Build script: `build.rs` captures `RUSTC_VERSION` at compile time for announce payload
- Announce payload uses `rustVersion` instead of `pythonVersion`
- All JSON timestamps use RFC 3339 UTC with trailing Z
- Dockerfile.transponder: multi-stage (rust:1.85-bookworm builder → debian:bookworm-slim runtime); dummy main.rs trick caches deps layer
- Entrypoint default: `/opt/transponder/bin/transponder` (was `.venv/bin/transponder` in Python era)
- Test suite: 39 tests across all modules; `cargo test` runs without external dependencies
- API tests: run with `VAULT_ADDR= python -m pytest api/tests/ -v` (unset Vault vars to use local dev keys)
- Telemetry storage layout: `data/telemetry/{YYYY-MM-DD}/{serviceName}/{environment}/{type}.jsonl`
- Telemetry store writes are fire-and-forget (log errors, don't block response)
- Retention walks date directories, prunes based on `ARECIBO_RETENTION_DAYS` (default 180)
- Query API follows Grafana JSON datasource conventions: GET with query params, not POST
- Schema pattern: schemas versioned as `{type}.1.0.0.json` with `$id` for naming, `$ref` for composition
- Query reader uses partition-aware reads: date range skips irrelevant dirs, service/env filtering at directory level
- Pagination cursor: base64-encoded JSON offset `{"o": N}`, opaque to clients
- Grafana: uses Infinity datasource plugin for REST API queries (yesoreyeram-infinity-datasource)
- Grafana provisioning: datasources in `grafana/provisioning/datasources/`, dashboards in `grafana/dashboards/`
- Docker compose Grafana: binds 127.0.0.1:3033, depends_on arecibo-api healthy, installs plugins via GF_INSTALL_PLUGINS
- Grafana sub-path proxy: GF_SERVER_SERVE_FROM_SUB_PATH=true with root_url ending in /arecibo/grafana/; anonymous Editor role since auth is proxy-managed
- Grafana proxy access control: isGrafanaMutationPath() targets dashboard/folder/org/admin mutation APIs; viewer-safe POSTs (e.g. /api/ds/query) are allowed
- WebSocket proxy: server.on('upgrade') with manual cookie parsing + Stytch session auth; pipes sockets bidirectionally via http.request upgrade

## 2026-03-03: arecibo-00005 - Runtime loop with lifecycle and shutdown

### Summary
Implemented TransponderRuntime with bootstrap (collector probing, announce, policy fetch), main loop (heartbeat/policy refresh/event flush on intervals, 200ms tick), directive handling (GO_DARK/RESUME/REFRESH_POLICY/SET_HEARTBEAT_INTERVAL/FLUSH_STATS), and graceful SIGTERM/SIGINT shutdown via ctrlc crate.

### Files Changed
- transponder/src/runtime.rs
- transponder/src/main.rs

### Learnings
- Counter lock must be dropped before HTTP calls and before apply_directives (which may call refresh_policy)
- Re-queue on failed event batch preserves Python behavior (events go back through push() which increments received counter)
- ctrlc crate handles both SIGTERM and SIGINT with a single handler via AtomicBool
